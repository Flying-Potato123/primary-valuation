## 1. VC Method（风险资本法）

### 1.1 适用场景

- 种子轮到 C 轮
- 公司尚未产生稳定利润或正现金流
- 有较清晰的退出路径（IPO 或 M&A）
- 投资者追求目标回报倍数（MOIC）

### 1.2 核心公式

```
Step 1: 预测退出价值（Exit Equity Value）
  Exit Equity Value = Exit Metric × Exit Multiple
  
  其中：
  - Exit Metric = 退出年份的 Revenue / EBITDA / Net Profit
  - Exit Multiple = 可比上市公司/可比交易的倍数（需打折）
  - 退出年份通常为 3-7 年（天使轮取 5-7 年，B/C 轮取 3-5 年）

Step 2: 计算投资者要求的退出收益（Required Exit Proceeds）
  Required Exit Proceeds = Investment Amount × Target MOIC
  
  其中：
  - Target MOIC 按轮次取值：
    · 天使/种子: 8x - 15x（高风险，长持有期）
    · Pre-A/A: 5x - 10x
    · B 轮: 3x - 7x
    · C 轮及以后: 2x - 5x

Step 3: 计算所需持股比例（Required Ownership）
  Required Ownership = Required Exit Proceeds / Exit Equity Value
  
  注意：此处 Required Ownership 必须是小数形式（如 0.15 = 15%）
  如果 Required Ownership > 50%，标记为"不可投"（估值过高）

Step 4: 计算投后估值（Post-money Valuation）
  Post-money Valuation = Investment Amount / Required Ownership

Step 5: 计算投前估值（Pre-money Valuation）
  Pre-money Valuation = Post-money Valuation - Investment Amount
```

### 1.3 退出倍数的打折规则

一级市场公司缺乏流动性、规模和上市确定性，退出倍数需要在可比上市公司倍数基础上打折：

| 当前轮次 | 退出倍数折扣（vs 上市公司倍数） | 说明 |
|----------|------------------------------|------|
| 天使/种子 | 40% - 55% | 退出不确定性极高 |
| Pre-A/A | 50% - 65% | 商业模式初步验证 |
| B 轮 | 60% - 75% | 增长路径较清晰 |
| C 轮 | 70% - 85% | 接近上市条件 |
| Pre-IPO | 80% - 90% | 预计 1-2 年内上市 |

### 1.4 退出倍数的行业调整

不同行业的退出倍数基准不同（见 [industry-benchmarks.md](industry-benchmarks.md)），但通用规则是：

- 软件/SaaS：倾向 EV/Revenue 倍数（3x-15x 视增长率）
- 消费品牌：倾向 EV/Revenue 或 EV/EBITDA
- 制造业/硬件：倾向 EV/EBITDA 或 PE
- 生物医药：倾向 rNPV（不适用 VC Method）

### 1.5 v3实践：VC Method 概率加权法（推荐用于天使/种子轮）

v3 版本的核心改进是将 VC Method 与概率情景分析深度融合，避免单一确定性估值：

```
三种情景设置规则：

乐观情景（概率15-25%）：
  - 退出营收 = BP预测 × 1.25（上浮25%）
  - PS倍数 = 接近可比上市公司中位数（流动性折价较小）
  - 后续稀释率 = 取下限（如45%）
  - 适用于：产品大规模放量、进入省级采购目录、IPO高估值

基准情景（概率50-55%）：
  - 退出营收 = BP预测值
  - PS倍数 = 可比上市公司中位数 × 一级市场折价（天使轮取55-65%）
  - 后续稀释率 = 取中位（如55%）
  - 适用于：核心假设基本兑现，按计划推进

悲观情景（概率20-25%）：
  - 退出营收 = BP预测 × 0.60（下浮40%）
  - PS倍数 = 极端保守值（如1.0x，接近清算价值）
  - 后续稀释率 = 取上限（如65%）
  - 适用于：推广受阻、竞争加剧、融资环境恶化

概率加权计算：
  概率加权MOIC = 乐观MOIC × P(乐观) + 基准MOIC × P(基准) + 悲观MOIC × P(悲观)
  概率加权退出估值 = 退出估值₁ × P₁ + 退出估值₂ × P₂ + 退出估值₃ × P₃
  概率加权投后估值 ≈ 概率加权退出估值 × (1 - 加权稀释率) / 概率加权MOIC
```

**关键参数**：
- 天使轮退出年份：5年
- 目标MOIC：≥8x（天使轮基准，政策鼓励赛道可下调至7x）
- 天使轮累计稀释（至IPO）：45%-65%
- 天使轮退出PS倍数：上市公司中位数 × 55-65%折价

### 1.6 常见错误

❌ 使用上市公司倍数不打折直接套用
❌ 退出年份假设过短（天使轮假设 3 年退出）
❌ Required Ownership > 50% 仍继续计算（应标记为不可投）
❌ 混淆 Enterprise Value 和 Equity Value
❌ 忽略退出时的净现金/净债务调整
❌ 只做单一情景估值不做概率加权（天使轮不确定性极高，必须做情景分析）

## 2. Scorecard Method（计分卡法）

### 2.1 适用场景

- 种子轮/Angel 阶段
- 几乎没有财务数据
- 需要快速评估相对于"行业平均"的溢价/折价

### 2.2 核心公式

```
Step 1: 确定行业基准估值（Baseline Valuation）
  Baseline = 同行业同阶段的平均投前估值（来自 industry-benchmarks.md）

Step 2: 对每个维度打分
  打分范围：0% - 200%（100% = 行业平均）
  
  维度及权重：
  | 维度 | 权重 | 说明 |
  |------|------|------|
  | 团队质量 | 30% | 创始人经验、团队完整性、行业背景 |
  | 市场规模 | 25% | TAM、增长率、市场结构 |
  | 产品/技术 | 15% | 技术壁垒、产品成熟度、IP 保护 |
  | 竞争环境 | 10% | 竞争强度、差异化、护城河 |
  | 市场/销售 | 10% | GTM 策略、渠道、客户获取 |
  | 其他因素 | 10% | 监管风险、退出环境、融资环境 |

Step 3: 计算加权得分
  Weighted Score = Σ (维度得分 × 权重)

Step 4: 计算估值
  Pre-money Valuation = Baseline × Weighted Score
```

### 2.3 打分指南

| 得分 | 含义 | 触发条件示例 |
|------|------|------------|
| 150%+ | 显著优于行业平均 | 连续创业者 + 百亿级 TAM + 专利技术 |
| 120%-150% | 优于行业平均 | 有经验的团队 + 大市场 + 差异化产品 |
| 80%-120% | 行业平均 | 一般团队 + 中等市场 + 有竞争 |
| 50%-80% | 弱于行业平均 | 首次创业者 + 小市场 + 红海竞争 |
| <50% | 显著弱于行业平均 | 团队不完整 + 市场萎缩 + 无差异化 |

### 2.4 注意事项

- 基准估值必须来自可靠的行业数据（标注来源）
- 每个维度的打分需要附简要理由
- Scorecard 通常**不作为独立估值方法**，而是与 VC Method 交叉验证

## 3. Berkus Method（伯克斯法）

### 3.1 适用场景

- 最早期阶段（无收入、无用户、甚至无产品）
- 种子轮/Angel 阶段
- 用于快速判断"这个想法值多少钱"

### 3.2 核心公式

```
Pre-money Valuation = Σ 各要素价值

五要素及价值上限（需根据行业和地区调整）：
| 要素 | 最高价值（人民币） | 说明 |
|------|-------------------|------|
| 合理创意 | 50万 - 100万 | 商业价值已验证的创意 |
| 原型产品 | 100万 - 200万 | MVP 或 Alpha 版本 |
| 优质团队 | 100万 - 300万 | 创始人+核心团队 |
| 战略关系 | 50万 - 150万 | 客户、渠道、合作伙伴 |
| 产品发布/销售 | 100万 - 250万 | 已有初步销售或用户验证 |

总计上限：500万 - 1000万 人民币（天使轮）
```

### 3.3 中国市场的调整

在中国市场，Berkus 方法的上限可以根据行业调整：

| 行业 | 总上限调整 | 原因 |
|------|----------|------|
| AI/大模型 | ×2.0 - ×3.0 | 热门赛道溢价 |
| 消费品牌 | ×1.0 | 标准 |
| 企业服务/SaaS | ×1.2 - ×1.5 | 经常性收入模式 |
| 生物医药 | ×1.5 - ×2.0 | 高门槛、长周期 |
| 硬件/制造 | ×0.8 - ×1.0 | 资本密集 |

## 4. PWERM（概率加权预期回报法）

### 4.1 适用场景

- B 轮及以后
- 退出路径多样化（IPO/M&A/继续运营/清算）
- 需要对不同情景做概率加权

### 4.2 核心公式

```
PWERM Value = Σ (Scenario i Value × Scenario i Probability)

典型情景（可按需扩展）：
| 情景 | 典型概率范围 | 价值计算方式 |
|------|------------|------------|
| IPO | 10% - 30% | 预测 IPO 市值 × (1 - IPO 折扣) |
| M&A | 20% - 40% | 可比 M&A 交易倍数 × 财务指标 |
| 继续运营 | 30% - 50% | VC Method 基准情景 |
| 清算/失败 | 10% - 30% | 清算价值或 0 |

约束条件：
- 所有情景概率之和 = 100%
- 每个情景的估值逻辑必须独立且合理
- 退出时间因情景而异（IPO 最久，M&A 次之）
```

### 4.3 概率设定的规则

- **不能拍脑袋**：每个概率需要基于"可以论证的因素"（团队质量、市场趋势、竞争格局、历史成功率）
- **保守原则**：如果没有强证据支持高 IPO 概率，默认不超过 15%
- **标注来源**：概率设定标注为"分析师判断"，附简要理由

## 5. OPM（期权定价法）

### 5.1 适用场景

- 有多个优先级不同的证券类别（普通股、多轮优先股）
- 退出价值不确定性高
- 需要精确计算各类股东的分配

### 5.2 核心概念

OPM 将各类股权视为公司整体价值的**看涨期权**，每个类别有一个行权价（breakpoint）。

```
基本步骤：
1. 确定公司整体股权价值（Enterprise Value - Net Debt）
2. 计算各 breakpoint（每个优先级的总清算优先权累计值）
3. 使用 Black-Scholes 模型计算每个"区间"的价值
4. 按各类股东在区间中的权益比例分配

参数：
- 行权价 = 累计清算优先权
- 波动率 = 可比上市公司波动率 × 1.5-2.0（一级市场溢价）
- 到期时间 = 预计退出年限
- 无风险利率 = 中国10年期国债收益率
```

### 5.3 何时使用 OPM vs PWERM

| 条件 | 推荐方法 |
|------|---------|
| 资本结构简单（仅普通股 + 1 轮优先股） | PWERM 即可 |
| 资本结构复杂（多轮优先股 + 不同条款） | OPM |
| 退出时间可以合理预测 | PWERM |
| 退出价值分布高度不确定 | OPM 更有优势 |

## 6. 可比交易法（Precedent Transactions）

### 6.1 适用场景

- 有活跃的一级市场融资数据
- 同行业近期有较多可比轮次

### 6.2 核心公式

```
估值范围 = 可比交易的估值倍数 × 标的公司对应指标

可比筛选条件（优先级从高到低）：
1. 同行业（一级分类相同）
2. 同轮次（允许 ±1 轮）
3. 相近时间（12 个月以内）
4. 相似规模（融资额 ±50%）
5. 相似地域（中国市场优先）
```

### 6.3 可比交易数据来源

按可靠性排序：
1. **公司公告/交易所披露**——最高可靠性
2. **PitchBook/Crunchbase/IT桔子**——中等可靠性
3. **36氪/媒体融资报道**——低可靠性，需标注"媒体口径"
4. **未确认的市场传言**——不纳入估值，仅做参考

## 7. DCF for VC（适用于中后期的简化 DCF）

### 7.1 适用场景

- B 轮及以后
- 至少有 2 年以上可验证的收入数据
- 增长趋势开始稳定

### 7.2 与二级市场 DCF 的关键差异

| 参数 | 二级市场 | 一级市场 |
|------|---------|---------|
| WACC | 7% - 12% | 15% - 30%（含创业失败风险） |
| 预测期 | 5-10 年 | 5 年（更长无意义） |
| 终值占比 | 60% - 80% | 50% - 70% |
| 终值增长率 | 2% - 3% | 2% - 4%（新兴行业可能更高） |
| 失败概率调整 | 通常不做 | **必须做**：每年乘以生存概率 |
| Beta | vs CSI 300 | 取可比公司 Beta 均值 × 1.3-1.5 |

### 7.3 风险调整 NPV（RA-NPV）

```
RA-NPV = Σ [FCF_t × Survival_Prob(t) / (1 + WACC)^t] 
       + Terminal Value × Survival_Prob(T) / (1 + WACC)^T

Survival_Prob(t) = Π(1 - Failure_Rate_i) for i = 1 to t

典型年化失败率假设（分析判断，标注来源）：
| 年份 | 天使轮 | A 轮 | B 轮 | C 轮 |
|------|--------|------|------|------|
| 第 1 年 | 25% | 15% | 8% | 4% |
| 第 2 年 | 20% | 12% | 6% | 3% |
| 第 3 年 | 15% | 8% | 4% | 2% |
| 第 4 年 | 10% | 5% | 3% | 1.5% |
| 第 5 年 | 7% | 4% | 2% | 1% |
```

## 8. 方法论混用与加权规则

### 8.1 加权原则

不同阶段的方法论权重分配：

| 阶段 | VC Method | Scorecard | Berkus | Comps | DCF | PWERM |
|------|-----------|-----------|--------|-------|-----|-------|
| Angel/种子 | 40% | 30% | 20% | 10% | — | — |
| Pre-A/A | 50% | 20% | — | 30% | — | — |
| B 轮 | 40% | — | — | 30% | 30% | — |
| C 轮 | 30% | — | — | 25% | 25% | 20% |
| Pre-IPO | — | — | — | 30% | 40% | 30% |

### 8.2 使用规则

1. **至少 2 种方法**交叉验证（不可单一方法）
2. 如果不同方法估值差异 >2x，需要**解释原因**并标注不确定性
3. 对早期公司，VC Method 和 Scorecard 权重合计不低于 60%
4. 如果某方法的 Required Ownership > 50%，**排除该方法**并说明原因
5. 最终估值范围取各方法估值范围的**交集或加权区间**，不取极端值

### 8.3 估值结论的呈现格式

```
最终估值建议：

投前估值区间：¥X - ¥Y 万
投后估值区间：¥X' - ¥Y' 万
基准情景（投前）：¥Z 万

估值方法贡献：
- VC Method（PS倍数，8x MOIC）：¥A 万（权重 40%）
- Scorecard Method（行业基准 120%）：¥B 万（权重 30%）
- 可比交易法（3 笔可比交易中位数）：¥C 万（权重 30%）

与 BP 隐含估值对比：
- BP 隐含投前估值：¥D 万
- 分析师估值中位数 vs BP：溢价/折价 X%
```
