# Project Analysis Playbook

Use this reference when the valuation object is a project, product line, business plan, acquisition case, private venture, or internal investment.

## Setup Questions

Clarify:

- Decision: launch, fund, acquire, continue, expand, abandon, or sell.
- Valuation date and currency.
- Project life: finite, renewable, or expected to become a going concern.
- Owner perspective: sponsor, minority investor, lender, employee, acquirer, or founder.
- Funding need and claim structure: common equity, preferred equity, debt, convertibles, options, earnouts.
- Available data: historical operations, market research, customer funnel, unit economics, cost budget, capex plan, taxes, working capital, and exit assumptions.

## Incremental Cash Flow Rules

Include:

- Incremental revenue, operating costs, taxes, capex, maintenance spend, working capital, and terminal/salvage value.
- Opportunity costs of assets, people, capacity, data, distribution, or brand used by the project.
- Cannibalization or spillover effects on the existing business.
- Required support costs, overhead that truly changes, and post-launch maintenance.

Exclude:

- Sunk costs.
- Allocated overhead that does not change.
- Financing flows when using unlevered project cash flows and WACC.
- Accounting earnings that do not map to cash flow.

## Project Story To Numbers

Map the story into drivers:

- Customer problem -> adoption curve, conversion, retention, churn, pricing.
- Market opportunity -> TAM/SAM/SOM, penetration, volume, utilization.
- Competitive edge -> margin, pricing power, market share persistence, fade period.
- Delivery model -> fixed costs, variable costs, capex, working capital, operating leverage.
- Execution risk -> launch delay, cost overrun, survival probability, milestone failure.
- Optionality -> expand, delay, abandon, pivot, license, or sell.

## Discount Rate And Currency

- Use the project risk, not the parent company's average risk, if the project is meaningfully different.
- Match currency of cash flows to risk-free rate and inflation assumptions.
- If local currency cash flows are forecast, use local currency discount rates or convert expected cash flows with consistent exchange-rate assumptions.
- If using WACC, use target capital structure appropriate for the project/business, not necessarily current book leverage.
- For venture-like projects, consider scenario-weighted values and dilution rather than a single inflated discount rate.

## Model Choices

- Capital budgeting NPV: best for finite projects with clear incremental cash flows.
- FCFF going concern: best when the project becomes a standalone business.
- Probability-weighted DCF: best for milestone projects with survival/failure branches.
- Decision tree: best for staged investment, regulatory approval, clinical trials, resource exploration, or launch gates.
- Real option: best when waiting, expanding, abandoning, or licensing is a real choice with exclusivity.
- Venture valuation cross-check: useful for funding rounds, but reconcile it back to intrinsic assumptions.

## Deliverable Template

Use this structure:

1. Executive conclusion: value range, decision, confidence, and biggest swing factor.
2. Project narrative: what has to be true for the project to work.
3. Assumptions table: source, base case, bear, bull, and rationale.
4. Incremental cash flow summary: revenue, costs, taxes, reinvestment, FCF.
5. Discount rate / required return build.
6. Terminal, exit, salvage, or abandonment logic.
7. Intrinsic value range and NPV/IRR/payback if relevant.
8. Sensitivities: top 3-5 drivers only.
9. Pricing/market cross-check: peers, transactions, or venture marks where relevant.
10. Decision triggers: milestones, kill criteria, expansion triggers, and what would change the answer.

## Minimum Viable Assumptions Table

For each key driver, capture:

- Driver name.
- Base value.
- Bear value.
- Bull value.
- Source or rationale.
- Why it matters.
- What evidence would update it.

Core drivers usually include price, volume/adoption, churn/retention, gross margin, operating margin, capex, working capital, tax, reinvestment, terminal growth/exit multiple, discount rate, survival probability, and dilution.

## Red Flags

- The project is attractive only because sunk costs are treated as recoverable.
- The model uses the parent WACC even though project risk is different.
- Revenue grows without capacity, customers, sales effort, or reinvestment.
- Margin expansion is assumed without a cost structure story.
- Terminal value assumes a mature company but the forecast never reaches maturity.
- Optionality is claimed without exclusivity or a real managerial choice.
- A venture project is valued with only a target exit multiple and no path to cash flows.
